TEST CASES

A suite of test cases has been established for quality assurance testing, as well as to provide examples for new users.

Each test case shall provide at least the following:

  • Brief description of the case
  • Graphics to illustrate the case
  • One or more grid files (tarred, compressed)
  • All necessary input files to run the case (tarred, compressed)
  • Expected results including run time, memory required, convergence history
  • The date and platform on which the results were obtained

In addition, comparisons with experimental data shall be made whenever possible.
